Independent Verification of Don't Waste Bits!:
Scale-Dependent INT4 Losslessness and a DWB-TurboQuant Extension

themoddedcube / LonghornSilicon
github.com/LonghornSilicon/dont-waste-bits · April 2026
Abstract We present an independent reproduction of "Don't Waste Bits!" [arXiv:2604.04722, CVPR 2026], which proposes adaptive per-token KV-cache quantization via a learned 3-layer MLP controller assigning {2,4,8,16}-bit precision to each token. Since original code is not yet public (expected June 2026), we re-implement from paper equations and identify seven methodological insights. Our key finding is that symmetric per-tensor INT4 KV quantization exhibits scale-dependent losslessness: at 135M/360M parameters (15 attention heads), INT4 ≈ FP16 (~0pp gap), but at 1.7B (32 heads) INT4 shows genuine ~10pp degradation matching the paper's stated baseline. The paper's 33.6% static INT4 baseline for smaller models is reproduced only with non-standard 8-level quantization (scale=max/3, equivalent to INT3), confirmed causally by ablation. We mechanistically verify this via εeff = εrel × rcancel: 8.1% at 360M (below threshold → lossless) vs. 12.4% at 1.7B (above threshold → 10pp loss). For H3, we find an implementation gap: our v1 achieves 33.8% (−7.4pp from paper), while v2 (better-trained) achieves 37.0% at 1.68× more bits — revealing a dual-objective tension. As a novel contribution, DWB-TurboQuant routes DWB's 2-bit tokens through PolarQuant (per-head Walsh-Hadamard rotation), recovering +2pp HellaSwag and +3pp ARC-Challenge at identical compression (5.05 avg bits).

1. Introduction

Large language models deployed on-device face a fundamental memory bandwidth bottleneck during autoregressive decoding: the KV cache grows linearly with sequence length and must be read from memory on every step. Quantizing the KV cache to lower bit-widths reduces this bottleneck at the cost of accuracy.

"Don't Waste Bits!" (DWB) [arXiv:2604.04722, CVPR 2026] proposes adaptive per-token quantization via a learned controller assigning {2,4,8,16}-bit precision per token. The paper reports for SmolLM-360M on HellaSwag: (H1) 17.75% latency reduction over static 4-bit KV; (H2) +7.60pp accuracy over static 4-bit KV; (H3) within 0.30pp of FP16. Original code is not yet public.

Contributions:

  2. Scale-dependent INT4 losslessness: mechanistic explanation via effective residual error across SmolLM-135M/360M/1.7B
  3. DWB-TurboQuant: routing DWB's 2-bit tier through PolarQuant, recovering +2–3pp at identical compression

2. Background

SmolLM model family. SmolLM-360M: 32 transformer layers, 15 attention heads, hidden dimension 960. SmolLM-1.7B: 24 layers, 32 heads, hidden dimension 2048.

HellaSwag evaluation. 4-way commonsense completion. The paper uses unnormalized log-likelihood accuracy (acc, ~42% for SmolLM-360M at FP16), not length-normalized (acc_norm, ~54%). This distinction is critical.

TurboQuant / PolarQuant. (ICLR 2026) Applies a per-head Walsh-Hadamard Transform rotation before uniform scalar quantization, decorrelating activations and reducing effective quantization error. The rotation is orthogonal and exactly invertible.

3. Re-Implementation

Controller: Linear(4,128) → ReLU → Linear(128,128) → ReLU → Linear(128,4) = 33,540 params. Signals: Ht (entropy), Rt (rarity), Vt (attn variance), Ct (confidence).

Training loss: L = α·CE + β·latency + γ·quality (Eq. 28), α=1, β=0.1, γ=0.1. Latency costs: [0.5, 1.0, 2.0, 4.0] for {2,4,8,16}-bit.

Critical finding (Insight 2): transformers ≥5.0 uses DynamicCache objects; hooks on attention output tuples silently fail. Fix: hook k_proj and v_proj Linear submodules directly (64 hooks for SmolLM-360M).

Finding: The dual-objective tension is not a hyperparameter issue. At β=0.1, avg_bits (5.30) ≈ paper's 5.05 — closest to the target — yet accuracy (39.0%) remains 2.2pp below paper's 41.2%. At β=0.5, avg_bits drops to 3.92 while accuracy is unchanged (39.0%) — higher penalty reduces bits with no accuracy benefit. At β≥1.0, the controller collapses entirely: every token gets 2-bit (avg_bits=2.00, bit_dist={2: 100%}), accuracy drops to 32.0%, and val_acc (0.257 / 0.227) falls below the random baseline of 0.25 — the latency penalty completely overwhelms the classification objective. No β simultaneously achieves both targets. End-to-end compound-loss training is qualitatively different from our quartile-classification approach.

6. DWB-TurboQuant (Novel Contribution)

Motivation: DWB assigns 42–57% tokens to scalar 2-bit (25% accuracy alone). PolarQuant (TurboQuant, ICLR 2026) applies per-head Walsh-Hadamard Transform rotation before scalar quantization, decorrelating activations and reducing effective error.

Method: Route DWB's 2-bit tier through PolarQuant instead of scalar INT2. Per-head WHT (head_dim=64 = 2⁶, power-of-2 ✓), exactly invertible, no extra bits.

7. Discussion

On scale-dependent losslessness. At 135M/360M, standard INT4 is already lossless — the paper's +7.6pp H2 gain is over a sub-standard baseline. At 1.7B, standard INT4 genuinely degrades and DWB provides real value. Practitioners should measure their model's effective residual error before assuming INT4 degrades meaningfully.

On the H3 gap. Our best reproduction (v2, 37.0%) is 4.2pp below the paper's 41.2%. The compound loss with explicit latency penalty is the key missing ingredient — it forces the controller to find the accuracy/compression Pareto front. Our β-sweep (Table 6) confirms this is fundamental: at β=0.1, avg_bits (5.30) nearly matches the paper's target but accuracy (39.0%) remains 2.2pp short; at β=1.0, the controller degrades to near-random (val_acc=0.257≈0.25) — there is no β in our framework that closes the gap. We expect the gap to close when original code releases (June 2026).

On DWB-TurboQuant. Routing low-importance tokens through WHT-based quantization provides a free accuracy improvement on generation and reasoning tasks. However, BoolQ shows a −20pp regression: WHT rotation is task-specific, benefiting open-ended generation while disrupting short binary-answer patterns. Bit-width assignment and within-tier quantization method are orthogonal dimensions — but the best within-tier method is distribution-dependent. Future work: per-task routing, per-tier vector quantization.

8. Conclusion

We re-implemented DWB from paper equations and conducted systematic verification across SmolLM-135M/360M/1.7B. Main finding: symmetric INT4 KV quantization is scale-dependent — lossless at ≤360M (eff. residual 8.1%) and genuinely lossy at 1.7B (12.4%), fully explained from first principles. The paper's H2 claim is most valid at 1.7B. For H3, a dual-objective tension prevents simultaneously achieving accuracy and compression targets with our re-implementation. DWB-TurboQuant exceeds the paper's DWB target on HellaSwag at identical compression.

Reproducibility: All code and data at github.com/LonghornSilicon/dont-waste-bits. Latency (H1) pending RTX 4090; arithmetic verification confirms 17.75% is self-consistent.
