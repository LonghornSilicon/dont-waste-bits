🎯 Critical Discovery: Metric Mismatch Resolved

Paper uses unnormalized log-likelihood (acc), not acc_norm

This was the root cause of all baseline discrepancies. Evidence:

  • Our acc_norm gave SmolLM-360M ≈ 49%, which equals the paper's SmolLM-1.7B FP16 score — clearly wrong metric
  • Direct test: acc (unnorm) on 50 validation samples = 42.0% vs paper's 41.5% ✓
  • PDF text: "Accuracy is computed based on the final multiple-choice answer selected by the model" — FP16 baseline taken from SmolLM paper ref [7]

FP16 baseline confirmed. Re-running KV-4bit and DWB conditions with corrected metric now.

KV Cache Hooks: Fixed for transformers 5.x

Original approach hooked attention module outputs (past_key_values), which is a DynamicCache object in transformers ≥5.0 — hooks silently found nothing to quantize, giving identical accuracy for all conditions.

Fix: Hook k_proj and v_proj Linear submodule outputs directly. These exist in all LLaMA-based models (SmolLM, Mistral, Qwen, etc.) and capture K/V tensors before attention computation — correct simulation of quantized KV cache storage.
