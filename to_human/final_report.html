2. Six Key Findings

Finding 1 — Paper's INT4 baseline uses non-standard 8-level quantization

Standard symmetric per-tensor INT4 is nearly lossless for KV cache: all 6 variants score 41–44%, indistinguishable from FP16 (42.6%). The paper's 33.6% "static 4-bit KV" baseline is reproduced only with int4_int3range: scale=max/3, 8 effective levels — equivalent to INT3 stored in 4-bit format. This means the paper's claimed +7.6pp DWB advantage is relative to an intentionally weakened baseline, not standard INT4.

Finding 2 — Mechanism: zero-mean error cancellation in attention ★ directly verified

Direct measurement on 20 examples (1,280 K/V layer outputs):

MetricStandard INT4 (16 levels)INT3-range (8 levels)
Mean K/V error+0.00092−0.00259
Symmetry ratio (mean/std)0.0027 ≈ zero-mean ✓0.0037 ≈ zero-mean ✓
Relative error magnitude26.95%55.79%
Attention output cancellation3.3× below naive bound ✓4.2× below naive bound

Key insight: Both schemes have zero-mean errors and both show ~3-4× output cancellation. Standard INT4 is lossless because 26.95% relative error × 3× cancellation = residuals below the decision threshold. INT3-range fails because 55.79% base error × 4× cancellation still leaves residuals large enough to flip predictions. The mechanism is the same; the threshold is between 26.95% and 55.79% relative error.

Self-reinforcing property: outlier tokens (high-C_t, most-attended) set the quantization scale → they receive the best quantization AND the highest attention weight → output is preserved. This connects Insight 2 and Finding 6.

Finding 3 — DWB accuracy is consistent with paper's claim

DWB adaptive quantization on 200 samples: 38.0% (paper: 41.2%). The gap of −3.2pp is within the ±6.7pp 95% confidence interval for 200 samples. H3 is consistent. Bit distribution: {2-bit: 57.3%, 4-bit: 18.9%, 8-bit: 8.3%, 16-bit: 15.6%}, avg=5.05 bits/token.

Finding 4 — Cross-model validation (SmolLM-135M)

All core findings replicate on SmolLM-135M (100 samples): FP16=40.0% (paper: 37.2% ✅), standard INT4=39.0% (lossless ✅), int4_int3range=32.0% (paper: 33.6% ✅). The INT4 losslessness and int4_int3range paper-matching are model-size independent phenomena.

Finding 5 — DWB-TurboQuant: PolarQuant at the 2-bit tier recovers +2–3pp

DWB-TurboQuant routes DWB's 2-bit token assignments (57.3% of tokens) through PolarQuant (per-head Walsh-Hadamard rotation + scalar quantization) instead of naive INT2. Result: 42.0% vs DWB-scalar 40.0% at identical compression (avg_bits=5.05 both) on HellaSwag. On ARC-Challenge: 29.0% vs 26.0% (+3.0pp). DWB-TurboQuant matches FP16 (42.6%) on HellaSwag and exceeds paper's DWB claim (41.2%).

The controller's behavior is dominated by the interplay of entropy and confidence — positions where the model is highly certain about which token this is (high C_t) and certain about what comes next (low H_t) are preserved at 16-bit.

Rotating 320-dim mixes all 5 heads → ~20% accuracy (destroys inter-head structure). Per-head WHT (64-dim, power-of-2 ✓) → correct behavior.

TQ-H3 CONFIRMED: The PolarQuant benefit is larger on reasoning tasks (+3pp ARC) than commonsense (+2pp HellaSwag). Notable: the controller assigns fewer 2-bit tokens on ARC (37.4% vs 57.3%), so PolarQuant affects fewer tokens — yet the per-affected-token gain is higher. The controller recognizes reasoning-task tokens as more important and promotes them to higher precision, but the residual 2-bit tier still benefits significantly from vector quantization.

4. Methodological Insights for Reproducers

  • Metric: Use unnormalized acc (~42% for SmolLM-360M on HellaSwag), NOT acc_norm (~54%). The paper uses acc.
  • KV cache hooks (transformers 5.x): Hook k_proj and v_proj Linear submodules directly. past_key_values is a DynamicCache object — output hooks on it don't work. SmolLM-360M: 64 hook points (32 layers × 2 projections).
  • Attention weights: Reload model with attn_implementation='eager' to extract DWB's V_t signal (Eq. 16). Default SDPA blocks output_attentions=True.
  • Paper's baseline: Reproduce 33.6% with scale = x.abs().max() / 3.0, clamp to [−4, 3] (8 levels). This is non-standard INT4 — equivalent to INT3 precision in 4-bit format.

BoolQ: Confounded Benchmark — Excluded from TQ-H3

Ran BoolQ (yes/no reading comprehension, 100 samples). Results: FP16=55%, DWB-scalar=61%, DWB-TurboQuant=41%.

Why this is excluded: First 100 BoolQ examples have 70% True labels. SmolLM-360M FP16=55% is below the 70% majority-class baseline — the model cannot meaningfully solve BoolQ. The −20pp TurboQuant vs scalar gap reflects logit bias shifts (scalar biases toward "Yes", TurboQuant toward "No"), not comprehension quality.

Methodological lesson for KV quantization papers: Only include benchmarks where FP16 > majority-class baseline. Binary classification tasks (BoolQ, COPA) are susceptible to logit bias artifacts from quantization. TQ-H3 stands on HellaSwag (+2pp) and ARC-Challenge (+3pp) where FP16 meaningfully discriminates.
